## 📚 조사 결과 요약

찾은 자료들은 크게 세 범주로 분류됩니다:

1. **실패 분류체계(Taxonomy) 논문** — 

`Failure Modes in LLM Systems`(Vinay, 2025)처럼 프로덕션 환경에서 LLM의 동작이 전통적 ML 모델과 근본적으로 다르며, 15가지 숨겨진 시스템 수준 실패 모드를 분류한 연구

와, 

MAST(Multi-Agent System Failure Taxonomy)처럼 150개 이상의 실행 추적을 분석해 시스템 설계 문제, 에이전트 간 불일치, 태스크 검증 등 3개 범주 14가지 실패 모드를 도출한 연구

가 핵심입니다.

2. **"모델 탓이 아니라 시스템 탓"을 직접 뒷받침하는 자료** — 

LLM 시스템 실패는 실제로 "모델 실패가 아닌 시스템 실패"이며, 오류가 여러 모듈 상류(upstream)에서 발생하기 때문에 프롬프트 엔지니어링이나 모델 튜닝만으로는 해결이 불가능하고 시스템 설계 수준에서 대응해야 한다

는 논거를 제공합니다.

3. **AI 프로젝트 실패의 구조적 원인 분석** — 

대부분의 AI 프로젝트 실패는 AI 기술 자체보다 전략, 사람, 데이터, 프로세스의 문제로 귀결된다

는 실증적 분석과, 

에이전틱 워크플로우의 실패가 이종(heterogeneous) 노드들을 가로질러 자연어 상호작용, 도구 호출, 동적 제어 로직을 통해 전파되기 때문에 실패 귀인과 수정이 특히 어렵다

는 구체적인 메커니즘 분석이 포함됩니다.

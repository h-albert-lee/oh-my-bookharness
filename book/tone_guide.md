# Tone Guide

## 기본 톤

- 한빛미디어 기술서 톤 지향
- 실무자가 공감할 수 있는 구체적이고 직접적인 서술
- 초급 독자를 고려한 명확하고 간결한 문체

## 금지 사항

- 선언문 스타일 금지 ("~해야 한다"의 반복적 나열)
- 과도한 수사·과장 금지
- 불필요한 영어 용어 남발 금지 (첫 등장 시 한글(영문) 병기 후 한글 사용)
- 코드 비중 최소화 (버전 이슈 고려, 이론 중심)

## 권장 사항

- 핵심 개념은 비유와 사례로 도입 (예: 코딩 에이전트의 샌드박스 = 하네스)
- 각 장의 도입부에서 독자가 왜 이 내용을 읽어야 하는지 동기부여
- 실제 프로젝트 경험에서 나온 구체적 상황 묘사
- 독자의 기존 SW 개발 경험과 연결하여 설명 (TDD → EDD 전환 등)

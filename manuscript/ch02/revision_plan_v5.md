## 1. Must Fix (반드시 수정)

### MF-01 | 중제목(###) 넘버링 추가
현재 넘버링 없는 ### 헤딩을 `### 2.M.K` 형식으로 변환:
- `### 비결정성은 버그가 아니다` → `### 2.3.1 비결정성은 버그가 아니다`
- `### temperature=0은 충분하지 않다` → `### 2.3.2 temperature=0은 충분하지 않다`
- `### 비결정성은 LLM만의 문제가 아니다` → `### 2.3.3 비결정성은 LLM만의 문제가 아니다`
- `### 첫 번째 지점: 정답 정의의 붕괴` → `### 2.4.1 첫 번째 지점: 정답 정의의 붕괴`
- `### 두 번째 지점: 비결정적 재현 불가능성` → `### 2.4.2 두 번째 지점: 비결정적 재현 불가능성`
- `### 세 번째 지점: 테스트 오라클의 붕괴` → `### 2.4.3 세 번째 지점: 테스트 오라클의 붕괴`
- `### 결정론적 컴포넌트는 여전히 TDD로 검증한다` → `### 2.4.4 결정론적 컴포넌트는 여전히 TDD로 검증한다`
- `### 이진 판정에서 스펙트럼 판단으로` → `### 2.5.1 이진 판정에서 스펙트럼 판단으로`
- `### 평가는 검증 가능하다` → `### 2.5.2 평가는 검증 가능하다`
- `### EDD의 정의` → `### 2.6.1 EDD의 정의`
- `### EDD의 전제와 TDD의 전제 비교` → `### 2.6.2 EDD의 전제와 TDD의 전제 비교`
- `### EDD의 세 구성요소` → `### 2.6.3 EDD의 세 구성요소`
- `### EDD는 TDD를 버리지 않는다` → `### 2.6.4 EDD는 TDD를 버리지 않는다`

### MF-02 | 잘못된 # 레벨 헤딩 수정
코드 블록 밖의 `# 의사 코드 — 실행 코드 아님`, `# 결정론적 시스템에서의 단위 테스트` 등이 # 레벨로 되어있다면, 코드 블록 내 주석이 아닌 경우 적절한 레벨로 조정하거나 제거한다.

### MF-03 | 중제목 없는 절에 중제목 추가
`## 2.1`과 `## 2.2`에 중제목이 없다면 최소 2개씩 추가:
- `### 2.1.1`, `### 2.1.2` ...
- `### 2.2.1`, `### 2.2.2` ...

### MF-04 | 구조물 형식 통일
blockquote 기반 구조물이 있다면 `::: box/note/tip` 형식으로 변환.

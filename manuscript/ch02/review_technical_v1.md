## Must Fix

- 75.76% 불일치율 수치 인용 방식 오류: 본문은 '약 4번 중 3번은 서로 다른 출력이 나왔다'고 해석하나, 75.76%는 4번 중 3번(75%)과 근사하게 맞지만, 원논문(web_03)의 정확한 실험 조건(모델명, 벤치마크, 측정 방법)이 본문에 명시되지 않아 수치가 맥락 없이 떠 있다. '동일한 문제에 대해 75.76%의 불일치율'이 무엇을 의미하는지(실행 쌍 간 불일치인지, 전체 문제 중 1회 이상 불일치 발생 비율인지)가 불분명하다. 원논문 확인 후 정확한 측정 단위와 실험 조건을 명시해야 한다.
- TDD 정립 시점 서술 오류: 본문은 'TDD는 1990년대 말에 켄트 벡이 익스트림 프로그래밍(XP)과 함께 정립한 개발 방법론'이라고 서술한다. 켄트 벡이 TDD를 XP의 일부로 대중화한 것은 사실이나, TDD의 독립적 서술·공식화는 2003년 저서 'Test-Driven Development: By Example' 출판으로 이루어졌다. '1990년대 말 정립'은 XP 실천법으로서의 초기 도입 시점이며, 방법론 정립 시점과 혼용될 경우 기술적으로 부정확하다. '1990년대 말 XP 실천법으로 등장하고 2003년 공식화된'으로 수정하거나, 시점 주장을 완화해야 한다.
- temperature=0 비결정성 원인 설명의 과장: 본문은 temperature=0에서도 비결정성이 발생하는 원인으로 'GPU 병렬 처리의 부동소수점 연산 순서 차이'와 '분산 추론 환경의 타이밍 차이'를 제시한다. 이는 기술적으로 타당하나, 실제로 많은 단일 GPU 환경에서 temperature=0 설정 시 높은 재현성이 확보된다는 반례가 존재한다. 75.76% 불일치가 분산 추론 또는 특정 클라우드 API 환경에서 발생한 수치인지 단일 환경 수치인지가 불분명하며, 이를 모든 배포 환경의 일반적 사실처럼 서술하는 것은 과장이다. '분산 추론 환경 또는 클라우드 API 호출 시'라는 조건을 명시해야 한다.

## Should Fix

- 전통적 ML 시스템(분류기, 추천 시스템)의 비결정성 설명 범위 과대화: '훈련 데이터의 순서, 초기화 방식, 하이퍼파라미터에 따라 동일한 입력에 대한 예측 결과가 달라질 수 있다'고 서술하나, 이는 추론(inference) 단계가 아닌 훈련(training) 단계의 비결정성이다. 훈련 완료 후 고정된 모델 가중치를 사용하는 추론 단계에서 전통적 분류기는 대체로 결정론적이다. LLM의 추론 단계 비결정성과 전통적 ML의 훈련 단계 비결정성을 동일선상에 놓으면 혼란을 야기한다. 두 유형의 비결정성이 발생하는 단계를 명확히 구분해야 한다.
- web_01(EDDOps, Xia et al., 2024) 인용이 각주로만 처리되고 본문 논증에 직접 활용되지 않음: Source Pack Notes는 이 논문을 'ch02 논증의 뼈대'로 지정하고 'TDD의 4가지 구조적 한계'를 명시적으로 참조할 것을 권장한다. 그러나 현재 초안의 2.4절은 3가지 지점만 제시하고 web_01의 분류 체계와 직접 대응하지 않는다. EDDOps 논문의 TDD 한계 분류와의 대응 관계를 본문에 명시하거나, 4가지 지점으로 보완해야 한다.
- '확률론(probabilism)'이라는 용어 사용 부정확: Chapter Brief는 'probabilism'을 사용하나, 이 문맥에서 정확한 대립 용어는 철학적 확률론(probabilism)이 아닌 '확률적(stochastic)' 또는 '비결정적(nondeterministic)'이다. 그림 2-1의 범례에서 '확률론적 시스템'은 'stochastic system'의 번역으로 적절하나, 본문 전체에서 'probabilism'이라는 철학 용어와 혼용될 경우 오해를 낳을 수 있다. 용어를 '확률적 시스템(stochastic system)'으로 통일하는 것이 기술적으로 더 정확하다.
- 번역가 비유의 정확성 한계: '기억을 지운 채 내일 다시 번역'하는 번역가 비유는 직관적이나, LLM의 비결정성은 기억 부재가 아니라 동일 모델이 동일 입력에서 샘플링 분포를 달리 실현하는 현상이다. 비유가 '무상태(stateless) 시스템에서의 다른 판단'보다 '망각'을 함의하여 독자에게 LLM이 이전 결과를 기억하지 못해 달라진다는 오해를 줄 수 있다. 비유를 보완하거나 비유 직후 '이 차이는 망각이 아니라 동일한 확률 분포에서 매번 다른 샘플이 추출되기 때문'임을 명시해야 한다.

# revision_plan.md — ch03 출판사 형식 리비전

## 배경
출판사(한빛미디어) 1차 피드백: 위계 넘버링 통일 및 구조물 형식 표준화 필요.
ch03은 절(##)과 중제목(###) 모두 넘버링이 없으며, 비표준 헤딩 형식 다수.

---

## 1. Must Fix (반드시 수정)

### MF-01 | 절(##) 헤딩에 넘버링 추가
- `## 도입` → `## 3.1 도입`
- `## 모델을 세 번 바꾼 팀` → `## 3.2 모델을 세 번 바꾼 팀`
- `## 실패는 어디에서 오는가 — 실패 지점 지도` → `## 3.3 실패는 어디에서 오는가 — 실패 지점 지도`
- `## 모델 귀인이 반복되는 이유` → `## 3.4 모델 귀인이 반복되는 이유`
- `## 시스템 관점이란 무엇인가 — 조작적 정의` → `## 3.5 시스템 관점이란 무엇인가 — 조작적 정의`
- `## [소결] PART 1이 말한 것: 하네스, EDD, 시스템` → `## 3.6 PART 1이 말한 것: 하네스, EDD, 시스템`
- `## [전환] 시스템의 구조를 이제 들여다볼 차례` → `## 3.7 시스템의 구조를 이제 들여다볼 차례`

### MF-02 | 중제목(###) 넘버링 추가
표/그림을 ###으로 사용하는 것은 부적절. 헤딩에서 분리:
- `### 그림 3-1. 실패 지점 지도(Failure Topology)` → 본문 내 `**그림 3-1. 실패 지점 지도(Failure Topology)**`로 변환 (헤딩이 아닌 캡션)
- `### 표 3-1.` → 본문 내 `**표 3-1.**`로 변환
- `### 표 3-2.` → 본문 내 `**표 3-2.**`로 변환

실제 중제목은 넘버링 추가:
- `### 원인 1: 가시성 비대칭` → `### 3.4.1 가시성 비대칭`
- `### 원인 2: AI 의인화 경향` → `### 3.4.2 AI 의인화 경향`
- `### 원인 3: 평가 기준의 부재` → `### 3.4.3 평가 기준의 부재`
- `### 역사적 선례: 조종사 탓에서 시스템 탓으로` → `### 3.4.4 역사적 선례: 조종사 탓에서 시스템 탓으로`
- `### 무엇을 다르게 보는가` → `### 3.5.1 무엇을 다르게 보는가`
- `### 무엇을 다르게 묻는가` → `### 3.5.2 무엇을 다르게 묻는가`
- `### 실천으로서의 시스템 관점 — 블레임리스 포스트모템` → `### 3.5.3 실천으로서의 시스템 관점 — 블레임리스 포스트모템`
- `### 규모의 문제가 아니다` → `### 3.5.4 규모의 문제가 아니다`

### MF-03 | 비표준 헤딩 마커 제거
`[소결]`, `[전환]` 같은 메타 마커를 헤딩에서 제거하고 내용으로 자연스럽게 처리.

### MF-04 | 중제목 부족 절에 추가
`## 3.1 도입`, `## 3.2 모델을 세 번 바꾼 팀`, `## 3.3 실패 지점 지도`에 중제목 최소 2개씩 추가.

### MF-05 | 구조물 형식 통일
blockquote 기반 구조물이 있다면 `::: box/note/tip` 형식으로 변환.

## 2. Rewrite Instructions for Writer

1. 모든 ## 헤딩에 `3.M` 넘버링을 추가한다.
2. 모든 ### 헤딩에 `3.M.K` 넘버링을 추가한다.
3. 표/그림을 헤딩(###)으로 사용하지 않고, 본문 내 캡션 형식으로 변환한다.
4. `[소결]`, `[전환]` 등 메타 마커를 제거하고 자연스러운 제목으로 대체.
5. 기존 내용은 최대한 유지. 형식 변환이 핵심.
6. 분량 목표 변경 없음 (18쪽, 약 18,000자).

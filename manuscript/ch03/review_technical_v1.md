## Must Fix

- 콜롬비아 우주왕복선 사고 연도 오류: 초안에서 '2003년 발사된 콜롬비아 우주왕복선'이라고 서술하나, 콜롬비아 우주왕복선(STS-107)은 1981년 첫 발사되었고 2003년 2월 1일에 귀환 중 공중분해(재진입 사고)로 해체되었다. '2003년 발사된'이 아니라 '2003년 사고가 발생한' 또는 '2003년 임무 중 사고를 당한'으로 수정해야 한다. 또한 해당 보고서에 '부품은 설계대로 작동했다. 시스템이 실패했다'는 정확한 인용구가 존재하는지 source pack에서 확인되지 않으며, 출처 미표기 인용구를 사실처럼 제시하는 것은 기술 서적에서 허용되지 않는다.
- Sculley et al.(2015) '5%' 수치의 맥락 오용 위험: 초안은 'ML 코드(모델 코드)는 전체 시스템에서 5% 이하를 차지한다'고 단정적으로 인용하나, 원 논문(Hidden Technical Debt in Machine Learning Systems)의 해당 표현은 그림의 비유적 표현으로, 특정 시스템에서 측정된 정확한 수치가 아니라 개념적 비율 강조를 위한 서술이다. '실제 측정 수치'처럼 제시하면 독자가 보편적 사실로 오해할 수 있으므로, '개념적으로 표현하면' 또는 '비유적으로'와 같은 완화 표현을 추가해야 한다.
- RAND 보고서 'AI 프로젝트 실패율 80%' 수치의 출처 검증 필요: source pack note에서 web_01(RAND 보고서)이 해당 수치의 출처로 지목되어 있으나, 초안 본문에는 각주([^web_01])만 표기될 뿐 보고서명, 발행연도, 정확한 제목이 제시되지 않는다. 기술서에서 80%라는 강한 통계 수치를 사용할 때는 보고서의 정확한 서지정보와 해당 수치가 어떤 모집단·방법론으로 산출되었는지를 밝혀야 한다. 현재 형태로는 독자가 수치를 검증할 수 없다.

## Should Fix

- GPT-3.5 → GPT-4 → Claude 교체 사례의 시제 및 버전 정합성: 팀 사례에서 'GPT-3.5에서 GPT-4로' 교체 후 'Claude를 도입'하는 흐름이 제시되는데, 이는 특정 시점의 모델 버전명을 구체적으로 명시한 가상 사례다. 가상 사례에 실제 제품명을 결합하면 독자가 해당 모델의 실제 성능 차이로 오해할 수 있다. 모델명을 일반화하거나(예: '모델 A → 모델 B'), 실제 익명 사례임을 명시하거나, 또는 가상 사례임을 분명히 표시하는 것이 기술 서적의 관례에 부합한다.
- Ma(2025) 인용의 수치 불명확성: '오케스트레이션과 런타임 계층에서 발생한 실패가 전체 장애의 적지 않은 비중을 차지한다'는 표현은 구체적 수치 없이 모호하게 서술되어 있다. source pack에 해당 수치가 있다면 명시하고, 없다면 '적지 않은 비중'보다 '일정 비율'이나 '유의미한 비율'처럼 표현 수준을 일관되게 맞추거나 해당 논문의 실제 수치를 인용해야 한다.
- Vinay(2025) '상당 비율' 표현의 수치 불명확성: 'Vinay(2025)는 LLM 특화 실패 모드 15가지를 분류하면서, 이 지점에 해당하는 파싱 실패와 형식 불일치가 전체 운영 장애의 상당 비율을 차지한다고 분석한다'고 서술하나, 실제 수치나 비율이 제시되지 않는다. source pack을 확인하여 해당 수치가 존재하면 명시하고, 없다면 '원인 중 하나로 분류된다'는 수준으로 표현 강도를 낮춰야 한다.
- Sculley et al.(2015)의 'boundary erosion'과 'hidden feedback loop' 번역의 정확성: 초안은 각각 '경계 침식'과 '숨겨진 피드백 루프'로 번역했다. 원 논문의 'boundary erosion'은 모듈 경계가 암묵적 데이터 의존으로 무너지는 현상을 지칭하는 것으로, 번역 자체는 수용 가능하나 해당 개념이 '가시성 비대칭'의 기술적 표현이라고 연결짓는 논리 도약이 있다. 원 논문에서 이 개념들은 기술적 부채(technical debt)의 맥락에서 제시된 것으로, '가시성 비대칭'과의 연결은 저자의 해석임을 명시해야 한다.
- Holden(2009) 인용의 맥락 검증 필요: source pack note에 web_13으로 표기된 Holden(2009)이 항공 사고의 귀인 패턴을 다룬다고 서술되어 있으나, 초안 본문의 '1970년대까지 항공 사고의 원인 대부분은 조종사 실수로 결론지어졌다'는 서술이 실제 해당 논문의 주장인지, 아니면 저자의 추론인지가 불명확하다. 인용 범위를 명확히 표시하고, 통계적 주장('대부분')의 근거를 해당 논문에서 찾아야 한다.

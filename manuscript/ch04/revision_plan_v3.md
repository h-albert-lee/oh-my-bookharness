출판사(한빛미디어) 1차 피드백: 위계 넘버링 통일 및 구조물 형식 표준화 필요.
ch04는 절(##)과 중제목(###) 모두 넘버링이 없으며, 장 제목에 장 번호가 빠져있음.

## 1. Must Fix (반드시 수정)

### MF-01 | 장 제목에 번호 추가
`# LLM 파이프라인 기본 구성 요소` → `# 4장. LLM 파이프라인 기본 구성 요소`

### MF-02 | 절(##) 헤딩에 넘버링 추가
- `## 도입: 어디서부터 생각해야 하는가` → `## 4.1 도입: 어디서부터 생각해야 하는가`
- `## 파이프라인이라는 관점` → `## 4.2 파이프라인이라는 관점`
- `## 입력 처리기: 시스템이 받아들이는 것을 통제하라` → `## 4.3 입력 처리기`
- `## 프롬프트 구성기(Prompt Builder): 맥락을 조립하는 단계` → `## 4.4 프롬프트 구성기(Prompt Builder)`
- `## LLM 호출기: 모델 호출을 격리된 단계로 두어야 하는 이유` → `## 4.5 LLM 호출기`
- `## 출력 처리기: 모델 응답을 시스템이 사용할 수 있는 형태로` → `## 4.6 출력 처리기`
- `## 상태 관리자(State Manager): LLM은 기억하지 못한다` → `## 4.7 상태 관리자(State Manager)`
- `## 오류 처리기(Error Handler): 실패를 예외가 아닌 설계 대상으로` → `## 4.8 오류 처리기(Error Handler)`
- `## 왜 분리하는가: 교체 가능성과 평가 가능성` → `## 4.9 왜 분리하는가: 교체 가능성과 평가 가능성`

### MF-03 | 잘못된 # 레벨 헤딩 수정
코드 블록 밖의 잘못된 # 레벨 헤딩:
- `# LLM 호출기 — 역할 분리 예시 (의사코드)` → 코드 블록 내 주석이면 유지, 아니면 제거 또는 적절한 레벨로 조정
- `# 출력 처리기 — 파싱 실패 처리 예시 (의사코드)` → 동일

### MF-04 | 중제목(###) 추가 및 넘버링
현재 ### 중제목이 거의 없음. 각 절에 최소 2개의 중제목을 추가:
- 예: `## 4.3 입력 처리기` 아래에 `### 4.3.1 입력 검증의 목적`, `### 4.3.2 입력 전처리 패턴` 등

### MF-05 | 구조물 형식 통일
blockquote 기반 구조물이 있다면 `::: box/note/tip` 형식으로 변환.

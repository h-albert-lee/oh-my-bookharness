# revision_plan.md — ch01 출판사 형식 리비전

---

## 배경
출판사(한빛미디어) 1차 피드백: 장–절–중제목 간 위계 구분이 명확하지 않으며 넘버링이 필요함.
참고사항/용어 정의/blockquote 구조물의 형식 통일 필요.

---

## 1. Must Fix (반드시 수정)

### MF-01 | 절(##) 헤딩에 넘버링 추가
현재 `## 그날의 데모는 완벽했다` 형식 → `## 1.1 그날의 데모는 완벽했다` 형식으로 변경.
전체 절 목록:
- `## 그날의 데모는 완벽했다` → `## 1.1 그날의 데모는 완벽했다`
- `## AI 프로젝트 실패의 구조적 패턴` → `## 1.2 AI 프로젝트 실패의 구조적 패턴`
- `## AI는 소프트웨어처럼 동작하지 않는다` → `## 1.3 AI는 소프트웨어처럼 동작하지 않는다`
- `## 데모가 성공한 진짜 이유` → `## 1.4 데모가 성공한 진짜 이유`
- `## "모델이 문제다" — 실패 귀인 오류의 패턴` → `## 1.5 "모델이 문제다" — 실패 귀인 오류의 패턴`
- `## 하네스: 비공식 구조를 시스템으로` → `## 1.6 하네스: 비공식 구조를 시스템으로`
- `## 장 요약` → `## 1.7 장 요약`

### MF-02 | 중제목(###) 추가 및 넘버링
각 절 안에 최소 2개의 중제목을 추가한다. 형식: `### 1.1.1 중제목`
현재 `### — 하네스의 부재` 같은 비표준 형식은 제거하고, 본문 내에서 자연스러운 하위 구조를 만든다.

### MF-03 | 구조물 형식 통일
현재 blockquote 기반 구조물을 ::: 형식으로 변환:
- `> **용어 정의 | 비결정성(Nondeterminism)**` → `::: note 비결정성(Nondeterminism) ... :::`
- `> **[참고] temperature=0은 비결정성을 제거하지 않는다**` → `::: box temperature=0은 비결정성을 제거하지 않는다 ... :::`

### MF-04 | 장 제목 형식 확인
`# 1장. AI는 왜 항상 데모까지만 잘 되는가` — 이미 올바른 형식. 유지.
`### — 하네스의 부재` 부제는 장 제목 바로 아래에 일반 텍스트로 처리하거나 삭제.

### MF-05 | 표/그림 캡션 형식
- `**표 1-1. 데모 환경과 운영 환경 비교**` — 올바른 형식. 유지.
- `**그림 1-1.**` — 올바른 형식. 유지.

## 2. Should Fix (권장 수정)

### SF-01 | 참고 자료 섹션 넘버링
`## 참고 자료` → `## 1.8 참고 자료` 또는 넘버링 없이 유지 (출판사 확인 필요).

## 3. Rewrite Instructions for Writer

이번 리비전의 핵심은 **내용 변경 없이 형식만 수정**하는 것이다.
1. 모든 ## 헤딩에 `1.M` 넘버링을 추가한다.
2. 각 절 안에 본문 흐름에 맞는 중제목(### 1.M.K)을 최소 2개씩 추가한다. 기존 내용을 논리적으로 분할하라.
3. blockquote 기반 구조물(`> **[참고]**`, `> **용어 정의 |**`)을 `::: box/note/tip` 형식으로 변환한다.
4. 기존 본문 내용은 최대한 유지하되, 중제목 삽입으로 인한 자연스러운 문단 조정은 허용한다.
5. 분량 목표는 변경 없음 (16쪽, 약 16,000자).
